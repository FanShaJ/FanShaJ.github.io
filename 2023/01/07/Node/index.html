1.node.js定义

node.js是开源的，跨平台的，JavaScript运行环境

2.node.js是一个框架吗？– 不是

组成：

  • ECMAScript
  • node的核心模块（fs，http, path等核心模块）

3.node.js 可以做什么

  • 写接口
  • 写爬虫
  • 写桌面应用程序 Electron

六 模块化

1. 概念

将大文件拆分成互相依赖的若干小文件 ，这些小文件还可以通过特定的语法组合到一起 ，此过程称之为模块化

2. 模块化规范

  • AMD
  • CMD
  • commonJS — node.js
  • ESM — ES6 Module

3. Node.js中的模块分类

  • 自定义模块
  • 内置模块
  • 第三方模块

七 同源&跨域

1. 同源

  • 同源指的是两个URL地址具有相同的协议、主机、端口号。⇒ (协议，域名，端口)
  • 同源策略（英文全称 Same origin policy）是浏览器提供的一个安全功能。

2. 跨域

Ⅰ 概念

两个URL地址的协议、主机、端口号 ⇒ (协议，域名，端口) ，其中任意一个不相同即为跨域

Ⅱ 解决方法（目前主流）

  • 代理服务器

  • CORS （Cross-Origin Resource Sharing），跨域资源共享

    • 通过设置一个响应头来告诉浏览器，该请求允许跨域，浏览器收到该响应 以后就会对响应放行

2. readyState & status

  • xhr.readyState ===> 表示的是xhr这个对象当前所处的一个状态，总是下列的某一种情况

  • 0 UNSENT 代理被创建，但尚未调用 open() 方法。

  • 1 OPENED open() 方法已经被调用。

  • 2 HEADERS_RECEIVED send() 方法已经被调用，并且头部和状态已经可获得。

  • 3 LOADING 下载中；responseText 属性已经包含部分数据。

  • 4 DONE 下载操作已完成。

  • xhr.status ===> http状态码

  • 请求成功的状态码

    xhr.status >= 200 && xhr.status < 300 || xhr.status === 304

3. xhr.load & xhr.onreadystatechage

  • xhr.onload ==> 其实就是 readySate === 4 的时候，才会调用执行
    • 一次请求，这个回调只会执行一次
  • xhr.onreadystatechange ==> 只要readystate，这个xhr的状态发生变化了，就会执行
  • 一次请求，这个回调会调用多次
